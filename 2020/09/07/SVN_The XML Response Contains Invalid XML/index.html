SVN_The XML Resonse Contains Invalid XML

问题

  • 早上开发人员反馈无法SVN无法拉取代码，但是可以更新代码
  • 拉取代码提示

  • 问开发人员获取的SVN地址后，本地拉取代码提示

解决

  • 根据报错网上搜索得知
    1.SVN代码路径有问题，注意浏览器中的路径和拉取代码的时候的路径差异
    2.本地清除SVN缓存

3.找到出错的文件，删除后重新上传
4.新建一个仓库，重新上传代码

  • 逐一尝试了上述1 2方法，重启SVN服务后，没有解决问题，查看SVN的nginx配置文件发现,nginx会把请求转发到本地的SVN端口的服务上，我拉取代码的时候直接用nginx转发后的端口可以拉取
  • 仔细想过SVN配置没有变更过，影响SVN服务的因素有限，nginx的access日志过大无法查看，error日志提示磁盘空间不足，删除不重要的文件后空间释放，问题解决
